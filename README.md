# ABL Club Operations — Azure SQL to Microsoft Fabric Pipeline

End-to-end analytics pipeline for a fictional Australian Baseball League (ABL)
club operation. An OLTP source in Azure SQL Database is mirrored into Microsoft
Fabric in near real time, refined through a medallion architecture
(Bronze / Silver / Gold) with PySpark, modelled as a star schema, and surfaced
in Power BI, with Microsoft Purview governance.

## Data disclaimer — this is NOT real ABL data

This repository is a technical demonstration of a data pipeline, not a real
analysis of the Australian Baseball League. To be explicit:

- All game results, batting statistics, ticket sales, attendance, and member
  records are SYNTHETIC, generated by a Python script (data/03_generate_data.py).
- Only team names, venue names, and player names come from public ABL
  information. They are attached to fabricated performance and commercial data
  purely to make the demonstration look realistic.
- No number in this repository reflects the real on-field or commercial
  performance of any team, venue, or player.
- Member names and emails are entirely fake (Faker-generated) and exist only to
  demonstrate PII handling and data governance.

The purpose is to showcase the engineering pipeline, not to report real results.

## Architecture

    SSMS -> Azure SQL Database (OLTP source)
                  | Fabric Mirroring (near real-time, CDC)
                  v
            Fabric Mirrored DB -> OneLake
                  | shortcut
                  v
            Lakehouse: Bronze -> Silver -> Gold (PySpark)
                  v
            Star schema -> Power BI + Purview governance

## How to run (source database)

Run in SSMS connected to the Azure SQL Database (not MySQL, not local):

1. data/01_create_tables.sql — create tables
2. data/02_seed_dimensions.sql — seed dimensions, build dim_date
3. data/03_data_inserts.sql — load players, members, and fact tables
4. data/05_update_real_names.sql — real venue and player names
5. Configure Fabric Mirroring, then data/04_simulate_live.sql to test live replication
